# VoxelProxy

Бесплатный аналог [takker.ru](https://takker.ru) — прокси-сервер для Minecraft 1.16.5, позволяющий двум клиентам одновременно играть на одном сервере под одним ником.

---

## Как это работает

VoxelProxy запускается локально и слушает порт `25565`. Оба клиента подключаются к нему, а он уже подключается к реальному серверу. Первый подключившийся клиент становится **активным** (его ввод отправляется на сервер). Второй — **пассивным** (получает все пакеты от сервера, но его ввод игнорируется).

```
[Чит-клиент] ────┐
                 ├──► [VoxelProxy] ──► [Minecraft-сервер]
[Легит-клиент] ──┘
```

- Пакеты от **активного клиента** → сервер
- Пакеты от **сервера** → оба клиента
- Позиция игрока синхронизируется между клиентами
- Транзакции инвентаря отслеживаются и синхронизируются
- При отключении активного клиента управление автоматически переходит ко второму

---

## Использование

1. Запустите `voxelproxy.exe`
2. Введите адрес сервера
3. Программа выведет локальный IP-адрес для подключения
4. **Сначала** подключите клиент с читами — он станет активным
5. **Затем** подключите легит-клиент

---

## Возможности

- **Автопереключение** — если активный клиент отключается, управление переходит ко второму без разрыва соединения с сервером
- **Синхронизация позиции** — неактивному клиенту отправляются корректирующие пакеты позиции, чтобы его состояние не расходилось с активным
- **Синхронизация транзакций** — транзакции инвентаря (packet `0x07`/`0x11`) отслеживаются для обоих клиентов и корректно воспроизводятся при переключении
- **Проверка обновлений** — при запуске автоматически проверяется наличие новой версии на GitHub; при обнаружении открывает ссылку на скачивание
- **Горячая клавиша ALT+F10** _(только Windows)_ — скрывает/показывает консольное окно прокси
- **DNS-резолвер** — поддерживает SRV-записи (`_minecraft._tcp`), A/AAAA-записи и прямые IP-адреса

---

## Ограничения

- Поддерживается только **Minecraft 1.16.5** (протокол 754). Для других версий используйте [ViaProxy](https://github.com/ViaVersion/ViaProxy)
- **Лицензионные (online-mode) серверы не поддерживаются** — используйте ViaProxy как промежуточный слой
- Ввод от легит-клиента игнорируется, пока подключён чит-клиент
- Оба клиента должны быть одной версии

---

## Советы

- Для лучшей стабильности запускайте прокси на отдельном ПК
- Можно безопасно закрыть один из клиентов — второй продолжит играть
- Если сервер требует лицензию, подключитесь через ViaProxy с настроенным аккаунтом

## Перенаправление трафика (Linux, iptables)

> Только для Linux. Требуются права root/sudo и наличие `iptables`.

Скрипт `iptables-route.sh` перенаправляет TCP-соединения на уровне ядра Linux.
Когда Minecraft-клиент подключается к указанному IP:PORT, ядро прозрачно перенаправляет
соединение на другой адрес — например, на локальный VoxelProxy.

**Сценарий использования:** клиент подключается к `10.0.0.1:25565`, ядро перенаправляет
на `192.168.1.1:25565` (VoxelProxy), VoxelProxy проксирует на реальный сервер.
В выводе `ss -tunlp` видно подключение к `10.0.0.1`, а не к `192.168.1.1`.

### Использование

**Сделать скрипт исполняемым (один раз):**
```bash
chmod +x iptables-route.sh
```

**Интерактивное меню:**
```bash
sudo ./iptables-route.sh
```

**Добавить маршрут** — трафик на `10.0.0.1:25565` → `192.168.1.1:25565`:
```bash
sudo ./iptables-route.sh add 10.0.0.1:25565 192.168.1.1:25565
```

**Удалить маршрут:**
```bash
sudo ./iptables-route.sh remove 10.0.0.1:25565 192.168.1.1:25565
```

**Показать активные маршруты:**
```bash
sudo ./iptables-route.sh list
```

**Очистить упоминания скрипта из истории терминала:**
```bash
sudo ./iptables-route.sh clean-history
```

> **Важно:** правила iptables сбрасываются при перезагрузке. Маршруты сохраняются в
> `/etc/voxelproxy-routes.conf`. Для автовосстановления используйте `iptables-persistent`
> или добавьте вызовы скрипта в `@reboot cron`.
